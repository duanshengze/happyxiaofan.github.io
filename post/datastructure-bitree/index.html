查找算法之二叉查找树

发表于 2015-10-31   |   分类于 算法与数据结构   |  

二叉查找树是也称为二叉排序树，设计这种数据结构的目的是希望在查找的过程能够快速的动态删除和插入，而且时间复杂度是O(logn)，算是一种比较理想的动态查找算法了。二叉查找树要么是一颗空树要么具有如下三个性质：1）如果左子树不为空，那么其左子树上的所有节点的值都小于根节点的值；2）如果右子树不为空，那么其右子树上的所有节点的值都大于根节点的值；3）其左右子树也是一棵二叉查找树。对一个数据集合进行二叉查找的首要步骤就是构建二叉查找树。下面我们以一个具体的例子，说明构建二叉查找树的具体过程：

比如有一个集合{62,88,58,47,35,73,51,99,37,93}，要对这些元素构建一个二叉查找树

  1. 首先是62，由于是第一个节点，所以直接作为根节点
  2. 接着是88，由于88大于根节点62，所以将88作为62的右孩子
  3. 然后是58,58小于根节点62，所以把58作为根节点62的左孩子
  4. 继续是47,47小于62，应该在62的左子树，然后碰到了已经是62左孩子的58，继续比较58和47，明显47小于58，于是将47作为58的左孩子
  5. 再就是35,35小于62，插入的位置应该是左子树，小于58，在58的左边，而35小于47，于是将35作为47的左孩子
  6. 然后是73，大于62，小于88，于是把73作为88这个节点的左孩子
  7. 还有51，小于62，小于58，但是大于47，于是把51作为47的右孩子
  8. 接着是99,99大于62，还大于88，所以把99作为88节点是右孩子
  9. 继续是37,37小于47，但是大于35，于是把37作为35的右孩子
  10. 最后是93,93大于88，小于99，于是把93作为99的左孩子结点

如果我们对这棵二叉查找树进行中序遍历，可以得到遍历序列为{35,37,47,51,58,62,73,88,93,99}，是排序的。既然是有序的我们就可以利用折半查找算法、插值查找算法或者斐波那契查找算法。请看实现代码：

代码清单1

/* 递归查找二叉查找树中是否存在key */
/* @param T: 二叉查找树 */
/* @param key: 要查找的元素 */
/* @param f: 指向T的双亲 */
/* @param *p: 如果查找成功，则指针p指向该元素结点，并返回true */
/* 如果查找失败，则指针p指向访问的最后一个元素结点，并返回false */
Status BiTree_Search(BiTree T,int key,BiTree f,BiTree *p)
{
    if(!T)                              /* 查找失败 */
    {
        *p = f;
        return FALSE;
    }
    else if(T->data == key)             /*  查找成功 */
    {
        *p = T;
        return TRUE; 
    }
    else if(T->data > key)              /* 从左子树中递归查找 */
    {
        return BiTree_Search(T->lchild,key,f,p);
    }
    else                                /* 从右子树中递归查找 */
    {
        return BiTree_Search(T->rchild,key,f,p);
    }
}

比如，我们要查找51，程序的执行过程是这样的：

  1. 二叉查找树T指向根节点62，key=51,62的双亲结点是NULL
  2. 由于根节点不为空，所以程序执行第一个elseif判断语句，62不等于51，所以执行下一步
  3. 显然51<62,所以程序会递归从左子树中查找
  4. 根节点是左孩子是58，key=51,f=null
  5. 58仍然不等于51，而且51小于58，程序继续以58为根节点，递归从58的左子树中查找51
  6. 58的左孩子是47，显然47仍然不等于51，但是51大于47，所以程序继续以47为根节点，递归从47的右子树中查找
  7. 47的右孩子是51，比较51与key的值，显然是相等的，于是指针p指向51这个节点，程序返回TRUE，查找结束

那么如何在一棵二叉查找树中插入一个元素呢？正如上面的过程一样，只需要找到其插入的合适位置就可以直接插入到二叉查找树中了，看代码：

代码清单2：

/* 插入元素至二叉查找树中 */
/* @param T: 二叉查找树 */
/* @param key: 要插入的元素结点 */
Status BiTree_Insert(BiTree *T,int key)
{
    BiTree p,s;
    if(!BiTree_Search(*T,key,NULL,&p))      /*  查找不成功 */
    {
        s = (BiTree)malloc(sizeof(BiTNode));
        s->data = key;
        s->lchild = s->rchild = NULL;
        if(!p)                              /* 指针p指向NULL,说明是一棵空树 */
        {
            *T = s;
        }
        else if(key < p->data)
            p->lchild = s;
        else
            p->rchild = s;
        return TRUE;
    }
    return FALSE;
}

具体的插入过程已经在上面说过了，下面我们看看如何删除二叉查找树的一个结点，删除结点需要分为三种情况：

  • 既无左子树也无右子树，就是叶子结点
  • 只有左子树或者只有右子树
  • 既有左子树又有右子树

我们先看第一种情况：如何删除一个叶子结点呢？可以看出，当删除叶子结点，并不会破坏二叉查找树的性质，所以可以直接删除

然后，我们看看既有左子树又有右子树的情况，比如我们要删除47，该怎么处理呢？

一种较好的处理思路就是用47的前驱或者后继作为47的替代者，这里47的前驱是37,后继是48，我们发现用37或者48作为47的替代者，二叉查找树的性质没有改变，所以我们就考虑如何找到待删除结点的前驱或者后继，然后把待删除的结点替换为其前驱或者后继就好了。


那么其过程可以用代码实现如下：

代码清单3：

/* 使用待删除结点的前驱作为其替换者 */
Status Delete(BiTree *p)
{
    BiTree q,s;
    if((*p)->rchild == NULL)                /* 右子树为空，重接左子树 */
    {
        q = *p;
        *p = (*p)->lchild;
        free(q);
    }
    else if((*p)->lchild == NULL)           /* 左子树为空，重接右子树 */
    {
        q = *p;
        *p = (*p)->rchild;
        free(q);
    }
    else                                    /* 既有左子树又有右子树 */
    {
        q = *p;
        s = (*p)->lchild;
        while(s->rchild)                    /* 转左子树，以左子树为根节点找到待删除结点的直接前驱 */
        {
            q = s;
            s = s->rchild;
        }
(*p)->data = s->data;/* 循坏结束，s已经是待删除结点p的直接前驱了，把结点的值替换成前驱的值 */
        if(q!=*p)
            q->rchild = s->lchild;      /* 把s的左子树接到q的右子树上 */
        else
            q->lchild = s->lchild;
        free(s);
    }
    return TRUE;

}

/* 删除二叉查找树中为key结点 */
Status Delete(BiTree *T,int key)
{
    if(!*T)                                 /*  如果是空树，那么删除失败 */
        return FALSE;
    else
    {
        if((*T)->data == key)
            return Delete(T);
        else if((*T)->data > key)
            return Delete(&(*T)->lchild,key);
        else
            return Delete(&(*T)->rchild,key);
    }
}

注意带Delete函数与插入函数有一些类似，差别在于删除算法执行的是Delete(BiTree *p)函数，该函数就是根据结点左右子树的情况重接其左子树或者右子树

下面我们看看删除47结点的代码执行过程：

  1. 首先将待删除结点p赋值给临时变量q，再把p的左孩子赋值给s，此时q指向47，s指向35
  2. 进入循环，找到p的前驱结点
  3. 循坏结束，s指向37，把s->data赋值给p->data，此时p->data变为37。s也就是p的直接前驱结点，q指向35，也就是s的直接前驱
  4. 如果p和q的指向不同，那么就把q->rchild赋为s->lchild,在这个例子中，显然p与q的指向不同，那么q的右孩子变为s的左孩子，也就是36。
  5. 最后执行free(s)，就是把s结点删除，这里也就是把37这个节点删除

二叉查找树小结

由于二叉查找树是以链式存储的，当插入或者删除某个结点时候，只需要修改其左右孩子的指针就行，所以效率是比较高的，而如果使用二叉查找树进行元素的查找，由于二叉查找树的深度有限，那么花去的时间也不会太多。二叉查找树的平均时间复杂度是O(logn)，最差的时间复杂度是O(n)，最好的情况就是查找根节点，时间复杂度是O(1)，性能还是很不错的。

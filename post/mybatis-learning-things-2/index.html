Mybatis学习笔记（二）

发表于 2015-08-21   |   分类于 Mybatis   |     |   本文总阅读量 次

Mybatis在使用mapper接口开发具有便捷的特点，省去了开发人员编写接口实现类的过程；使用Mybatis的输入输出映射的特性使复杂的业务逻辑可以通过简单的xml配置得以实现；使用Mybatis的动态sql大大减少了sql语句的冗余，提高了代码的可读性。除此之外，Mybatis还具有很多优良特性，包括高级映射、延迟加载、查询缓存以及逆向工程等，其中逆向工程可以让Mybatis自动生成一堆mapper.java文件和mapper.xml文件，基于Mybatis框架的开发使用这些自动生成的类可以省去人工开发的成本，开发人员可以修改或者添加需要的mapper文件以实现业务需求。

高级映射

一对一查询

以用户购买商品为例：从用户到订单的角度看，用户一次可以创建多个订单。从订单到用户的角度看，一个订单只能由一个用户创建（因为每个订单的订单号都是唯一标识的，不可能出现重复的情况）。那么，假如要根据订单信息查询创建订单的用户的信息，该如何操作呢？这实际就是一对一查询，根据订单信息查询到的信息只可能是一个。下面说说在Mybatis上实现一对一查询的基本过程：

使用resultType实现

由于查询结果既包含订单信息（订单Id唯一标识查询结果）又包含用户信息，而在现有的POJO中只能满足其中一个（只有订单类或者用户类），只需要在订单类中添加需要查询的用户信息的属性即可：

public class OrdersCustom extends Orders{
    String username;
    Sring sex;
    String address;
    //setter和getter方法
}

然后需要在xxxMapper.xml中配置sql语句：

<select id="findOrdersUser" resultType="com.rhwayfun.mybatis.po.OrdersCustom">
    select order.*,user.username,user.sex,user.address from order,user where order.user_id = user.id
</select>

接着在xxxMapper.java中编写接口以及方法：

public interface OrdersCustomMapper{
    //根据订单信息查询创建订单的用户信息
    public OrdersCustom findOrdersUser() throws Exception;
}

然后就是编写测试代码了：

@Test
public void testFindOrdersUser() throws Exception{
    //创建会话
    SqlSession sqlSession = sqlSessionFactory.openSession();

    //创建代理 对象
    OrdersMapperCustom ordersMapperCustom = sqlSession.getMapper(OrdersMapperCustom.class);

    //执行查询
    List<OrdersCustom> orders = ordersMapperCustom.findOrdersUser();

    System.out.println(orders);

    sqlSession.close();
}

需要注意的是由于在xml中resultType的类型为OrdersCustom类对象，查询结果的所有字段都会映射到该类中属性中，如果不在OrdersCustom.java中添加查询用户信息的属性会导致映射失败。

使用resultMap实现

使用resultMap实现的基本思路就是把查询结果中订单信息映射到订单类中，而查询到用户信息则则映射到用户类中，那么问题来了，statement中的parameterType只能接收一个参数，如何把结果集中的数据分别映射呢？在第一篇中简单提到了包装的POJO，就是在一个普通的java对象中有另一个普通java对象的属性。这里就是在订单类中在下面使用代码实现：

第一步仍然是xml文件：

<!-- type:把查询结果映射到的类型中 -->
<resultMap id="OrdersUserResultMap" type="com.rhwayfun.mybatis.po.Orders">
    <!-- 订单信息 -->
    <id column="id" property="id"/>
    <result column="user_id" property="id"/>
    <result column="number" property="number"/>
    <result column="createtime" property="createtime"/>
    <result column="note" property="note"/>
    <!-- 用户信息
        property:订单类中包装的pojo类型
        javaType：上面类型中的全路径
     -->
    <association property="user" javaType="com.rhwayfun.mybatis.po.User">
        <id column="user_id" property="id"/>
        <result column="username" property="username"/>
        <result column="sex" property="sex"/>
        <result column="address" property="address"/>
    </association>
</resultMap>
<select id="findOrdersUserResultMap" resultMap="OrdersUserResultMap">
    select order.*,user.username,user.sex,user.address from order,user where order.user_id = user.id
</select>

接下来就是xxxMapper.java文件了：

//使用resultMap根据订单信息关联查询用户信息
public List<Orders>  findOrdersUserResultMap() throws Exception;

然后是测试代码：

@Test
public void testFindOrdersUser() throws Exception{
    //创建会话
    SqlSession sqlSession = sqlSessionFactory.openSession();

    //创建代理 对象
    OrdersMapperCustom ordersMapperCustom = sqlSession.getMapper(OrdersMapperCustom.class);

    //执行查询
    <u>List<Orders> ordersUser = ordersMapperCustom.findOrdersUserResultMap();</u>

    System.out.println(ordersUser);

    //释放资源
    sqlSession.close();
}

resultMap和resultType使用小结

resultType：使用resultType实现较为简单，如果pojo中没有包括查询出来的列名，需要增加列名对应的属性，即可完成映射。如果没有查询结果的特殊要求建议使用resultType。

resultMap：需要单独定义resultMap，如果对查询结果有特殊的要求，使用resultMap可以完成将关联查询映射pojo的属性中。

多对多查询

这里省略了一对多的情形，在多对多的情形中将会考虑这种情况。仍然以用户商品为例：如果要查询用户及其购买的商品信息，那这种情形就是典型的多对多的查询，针对多对多的情形往往要分解成一对多或者一对一的情形进行分析————因为一个用户可以购买多个商品，一个商品也可以被多个用户购买，而在数据库表中，用户表和商品表是没有直接关联的，把用户表和商品表连接起来的就是订单表和订单明细表————一个用户可以一次下多个订单，但是一个订单只能由一个用户确定；一个订单可以有多个订单明细（用户购买的商品列表），而一个订单明细只能属于一个订单；一个订单明细可以包含多个商品，一个商品也可以包含在多个订单明细中，所以订单明细和商品之间是多对多的关系。

根据用户————订单（一对多）————订单明细（一对多）————商品（一对一)就把多对多的关系分解成了一对多的关系，然后可以利用一对多关系的表之间的外键约束确定表与表之间的关联。

首先仍然是xml文件的配置：(使用resultMap)

<resultMap id="UsersAndItemsResultMap" type="com.rhwayfun.mybatis.po.User">
    <!-- 用户信息 -->
    <id column="user_id" property="id"/>
    <result column="username" property="username"/>
    <result column="sex" property="sex"/>
    <result column="address" property="address"/>
    <!-- 订单信息 -->
    <!-- 一个用户可以有多个订单 -->
    <collection property="ordersList" ofType="com.rhwayfun.mybatis.po.Order">
        <id column="orders_id" property="id"/>
        <result column="user_id" property="id"/>
        <result column="number" property="number"/>
        <result column="createtime" property="createtime"/>
        <result column="note" property="note"/>
        <!-- 订单明细信息 -->
        <!-- 一个订单包含有多个订单明细 -->
        <collection property="orderdetails" ofType="com.rhwayfun.mybatis.po.Orderdetail">
            <id column="orderdetails_id" property="id"/>
            <result column="items_id" property="itemsId"/>
            <result column="items_num" property="itemsNum"/>
            <result column="orders_id" property="ordersId"/>
            <!-- 一个订单明细对应一个商品 -->
            <association property="items" javaType="com.rhwayfun.mybatis.po.Items">
                <id column="item_id" property="id"/>
                <result column="item_name" property="name"/>
                <result column="item_detail" property="detail"/>
                <result column="item_price" property="price"/>
            <association/>
        </collection>
    </collection>
</resultMap>
<select id="findUsersAndItemsResultMap" resultMap="UsersAndItemsResultMap">
    select order.*,user.username,user.sex,user.address,orderdetails.orderdetails_id,orderdetails.items_id,orderdetails.items_num,orderdetails.orders_id,item.id item_id,item.name item_name,item.detail item_detail,item.price item_price from user,order,orderdetails,item where user.id = order.user_id and order.id = orderdetaisl.orders_id and orderdetails.items_id = item.id
</select>

接着是mapper接口的编写：

public List<User> findUsersAndItemsResultMap() throws Exception;

然后是测试类：

@Test
public void testFindOrdersUser() throws Exception{
    //创建会话
    SqlSession sqlSession = sqlSessionFactory.openSession();

    //创建代理 对象
    OrdersMapperCustom ordersMapperCustom = sqlSession.getMapper(OrdersMapperCustom.class);

    //执行查询
    List<User> userItems = ordersMapperCustom.findUsersAndItemsResultMap();

    System.out.println(userItems);

    //释放资源
    sqlSession.close();
}

查询缓存

查询缓存就是把数据先缓存起来，当下次执行同样的操作时就直接从缓存中取出数据，然后返回结果，从而提高系统性能，加快查询速度。Mybatis的查询缓存分为一级缓存和二级缓存：
一级缓存指的sqlsession级别的缓存，从前面的学习中可以知道，sqlsession都是定义在单个方法中的，相当于java中局部变量，所以别的sqlsession是不能操作此sqlsession的
二级缓存指的mapper级别的，多个sqlsession操作同一个mapper，所以二级缓存是支持sqlsession共用的

一级缓存

仍以查询用户信息为例：第一次查询的指定查询的用户id为1，Mybatis会先去缓存中查找是否有id为1的用户信息，如果没有就会执行sql语句查询从数据库中获取，并把查询结果放在缓存中；第二次仍然插叙id为1的用户信息，从缓存中查找，这次找到了，则直接返回该用户的信息。

如果程序中有commit()操作，则会清空缓存

下面编写程序测试一级缓存：

@Test
public void testCache() throws Exception{
    SqlSession sqlSession = SqlSessionFactory.openSession();
    UserMapper userMaper = sqlSesion.getMapper(UserMapper.class);

    //第一次查询
    User user1 = userMapper.findUserById(1);
    System.out.println(user1);
    //更新user1的信息
    user1.setName("张三");
    userMapper.updateUser(user1);
    //提交事务，执行后会清空缓存
    //sqlSession.commit();

    //第二次查询
    User user2 = userMapper.findUserById(1);
    System.out.println(user2);

}

如果没有注释sqlSeesion.commit()，可以从日志中发现当执行第二次查询的时候mybatis并没有从数据库查询，而是直接输出查询结果：这说明mybatis会先从缓存中查找数据，只有当缓存中没有相关数据的时候才从数据库中查找。

二级缓存

二级缓存与以及缓存的区别在于其范围更大，多个sqlsession对象可以共享二级缓存区域，二级缓存的原理与一级缓存相同，在于范围不同罢了。
二级缓存是默认关闭的，需要使用缓存要打开缓存开关（SqlMapConfig.xml）：

<setting name="cachedEnabled" value="true"></setting>

然后在xxxMapper.xml中开启缓存开关：

<cache/>

下面编写测试代码：

@Test
public void testCache2() throws Exception{
    //创建多个sqlsession对象
    SqlSession sqlSession1 = SqlSessionFactory.openSession();
    SqlSession sqlSession2 = SqlSessionFactory.openSession();
    SqlSession sqlSession3 = SqlSessionFactory.openSession();
    //创建不同sqlsession的代理对象
    UserMapper userMapper1 = sqlSession1.getMapper(UserMapper.class);
    UserMapper userMapper2 = sqlSession2.getMapper(UserMapper.class);
    UserMapper userMapper3 = sqlSession3.getMapper(UserMapper.class);
    //第一次查询id为1的用户
    User user1 = userMapper1.findUserById(1);
    System.out.println(user1);
    //关闭sqlsession1，执行这个操作会把数据写到二级缓存区域中
    //第二个sqlsession更新用户信息，findUserById(1)会先从二级缓存中获取数据
    User user2 = userMapper2.findUserById(1);
    user2.setName("李四");
    //提交事务，清空二级缓存
    sqlSession2.commit();
    //关闭会话
    sqlSession2.close();
    //第二次查询id为1的用户信息，由于第二个sqlsession执行了commit操作，二级缓存的//数据被清空，所以这次查询会从数据库中查询数据
    User user3 = userMapper3.findUserById(1);
    System.out.println(user3);
    sqlSession3.close();
}

mybatis和spring整合

将mybatis和spring整合的目的是使mybatis的SqlSessionFactory和mapper交给spring容器管理，mybatis和spring整合生成代理对象，SqlSessionFactory创建SqlSession都可以由整合后自动完成。整合后原始dao开发以及mapper接口开发的方法都有不同，请看：

原始dao开发

首先仍然是xxx.xml文件，这一步与原始dao开发相同，然后是接口，接口也与原始dao开发一致，接着是接口实现类，实现类除了要实现自己编写的接口还要继承SqlSessionDaoSupport类：

public class UserDaoImpl extends SqlSessionDaoSupport implements UserDao{
    public User findUserById(int id) throws Exception{
        //从SqlSessionDaoSupport继承而来
        SqlSession sqlSession = getSqlSession();
        User user = sqlSession.selectOne("test.findUserById",1);
        return user;
    }
}

接着在spring的配置文件中配置dao：

<!-- 加载配置文件 -->
<context:property-placeholder location="classpath:db.properties" />
<!-- 数据源，使用dbcp -->
<bean id="dataSource" class="org.apache.commons.dbcp.BasicDataSource"
    destroy-method="close">
    <property name="driverClassName" value="${jdbc.driver}" />
    <property name="url" value="${jdbc.url}" />
    <property name="username" value="${jdbc.username}" />
    <property name="password" value="${jdbc.password}" />
    <property name="maxActive" value="10" />
    <property name="maxIdle" value="5" />
</bean>
<!-- sqlSessinFactory -->
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <!-- 加载mybatis的配置文件 -->
    <property name="configLocation" value="mybatis/SqlMapConfig.xml" />
    <!-- 数据源 -->
    <property name="dataSource" ref="dataSource" />
</bean>
<!-- 原始dao -->
<bean id="userDao" class="com.rhwayfun.ssm.dao.UserDaoImpl">
    <property name="sqlSessionFactory" ref="sqlSessionFactory"><property/>
</bean>

然后就是测试程序了：

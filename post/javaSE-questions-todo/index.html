3、equals与==的区别
简单来说，1、基本类型的比较：使用==进行值的比较，如果使用了包装类，那么使用equals比较的仍然是值；2、引用类型的比较：在重写equals方法的前提下，equals比较的是内容是否相同，如果没有重写equals方法，则默认使用==进行比较，这就是说默认比较的是两个对象的地址。在Java中，Integer等包装类、String、Date都重写了equals方法。

//Object类的equals方法

    return (this == obj);
}

4、Object有哪些公用方法？
既然是公用方法，那么只有hashCode()、equals()、toString()、wait()、notify()、notifyAll()

hashCode():返回一个对象的哈希值，可以注意到，当一个对象多次调用hashCode()方法该对象返回的哈希值始终是相等的。
equals():默认进行对象地址的比较，只有new对象返回的地址相同才返回true
toString():返回一个对象的字符串表示，一般来说，toString()方法返回的值等同于getClass().getName() + ‘@’ + Integer.toHexString(hashCode())

wait()、notify()、notifyAll()是多线程中知识，这里暂不展开陈述

5、Java的四种引用以及用到的场景
强引用：具有强引用的对象，在内存不足时Java虚拟机宁愿抛出OutOfMemoryError也不会回收该对象。
软引用：如果一个对象只具有软引用，那么当内存足够时不会被回收，当内存不够的时候则会被回收。只要没有被回收，程序中就继续使用该对象。软引用主要用于内存敏感的高速缓存。软引用可以与一个引用队列一起使用，如果软引用的对象被回收，那么就会把该对象加入到关联的引用队列中
弱引用：与软引用类似，区别在于弱引用拥有更短暂的生命周期，当垃圾回收器扫描到它所管辖的区域中有只具有弱引用的对象，则不管内存是否足够都会将该对象及进行回收。弱引用也可以与一个引用队列一起使用，在弱引用的对象被垃圾回收后，垃圾回收器会把这个对象加入到其关联的引用队列中
虚引用：虚引用就是可有可无，与前两种引用不同，虚引用没有生命周期，即只具有虚引用的对象可能在任何时候被回收。虚引用必须于一个引用队列关联，主要作用是跟踪被垃圾回收对象的活动

6、Hashcode的作用
简单来说，hashCode主要用于解决存储大量数据查找效率的问题。就是说，hashCode根据一定的规则将对象的信息（比如存储地址、字段的值）映射成一个数值，这个数值就是哈希值。那么这个哈希值是如何发挥其强大作用呢？详细请参考我的这篇博客

7、ArrayList、LinkedList、Vector的区别
从底层数据结构看：ArrayList和Vector使用的是数组，LinkedList使用的是双向链表；
从执行效率看：ArrayList和Vector查找指定索引的元素效率很高，平均只需要O(1)的时间复杂度，而LinkedList由于是链表，查找效率较低，查找一个元素的平均复杂度是O(n)，但是LinkedList插入和删除元素相比ArrayList和Vector具有较高的效率，平均复杂度是O(1)
从线程安全的角度对比：Vector是线程安全的，ArrayList和LinkedList不是线程安全的；
从应用场景角度看：ArrayList应用于不需要同步且查找频繁的情况，Vector应用于需要同步且查找频繁的情况，LinkedList应用于插入与删除频繁的情况

8、String、StringBuffer与StringBuilder的区别
从字符的可变性考虑：String是不可改变的，StringBuffer与StringBuilder是可改变的；
从线程安全看：String与StringBuilder不是线程安全的，StringBuffer是线程安全的；
从执行效率考虑：StringBuilder > StringBuffer > String，简单分析一下，首先由于String是字符串常量，每次的字符串拼接都等同于创建新的对象，当拼接很多的时候，就会存在很多无效的引用，这时候GC会开始工作，从而导致效率降低，而StringBuilder和StringBuffer则是在原来的字符串对象进行修改，并不会产生新的对象，所以效率当然大于String。接下来说说为什么StringBuilder > StringBuffer？主要是由于StringBuffer是线程安全的，使用同步必要导致更大的资源开销，所以StringBuilder > StringBuffer。
从使用场景看：少量字符串拼接使用String，单线程大量字符串拼接使用StringBuilder，多线程大量字符串拼接使用StringBuffer

9、Map、Set、List、Queue、Stack的特点与用法
这个问题比较杂，基本牵涉了Java集合框架中的所有知识点，如果在面试中遇到这个问题，我觉得可以这么回答：Set、List、Queue、Stack都实现了Collection接口，属于Collection接口阵营，用于存放单个元素，Map是一种存放键值对元素的接口；Set中存放的元素是不可重复的，而且元素的存放没有顺序。List集合中存放的元素是可重复的，存放的元素是有序的。Queue类似生活中排队模型，实现的是数据结构中的“队列”这种数据结构，队列是一种先进先出（FIFO）的数据结构，每次都是队头出队，从队尾入队，要注意的是Queue不支持随机访问元素，只能从队头或者队尾进行操作。不过在Java集合框架中，Deque双向了一种“双端队列”，支持从队头和队尾添加删除元素。Stack实现了“栈”这种数据结构，栈具有先进后出（FILO）的特点，Stack类继承于Vector类，所以Stack是线程安全的

10、HashMap和HashTable的区别
第1点：HashMap是从jdk1.2之后才有的，从Hashtable从jdk1.0中就有，Hashtable是Dictionary的子类，同时也是Map接口的实现类，而HashMap是AbstractMap的子类同时也是Map接口的实现类
第2点：Hashtable不允许null作为key或者value，而HashMap允许
第3点：Hashtable是线程安全的，HashMap不是线程安全的
第4点：HashMap取消了Hashtable的contains()方法，增加了containsKey()和containsValue()方法
第5点：Hashtable的初始容量是11，每次增加的方式是oldCapacity*2 + 1，而HashMap的初始容量是16，且只能为2的指数次方大小。
第6点：哈希值不同。Hashtable直接使用hashCode方法返回的值，HashMap使用的是与运算重新计算哈希值

//HashMap计算哈希值的方法
final int hash(Object k) {
    int h = hashSeed;
    if (0 != h && k instanceof String) {
        return sun.misc.Hashing.stringHash32((String) k);
    }
    h ^= k.hashCode();
    // This function ensures that hashCodes that differ only by

    // number of collisions (approximately 8 at default load factor).
    h ^= (h >>> 20) ^ (h >>> 12);
    return h ^ (h >>> 7) ^ (h >>> 4);
}

11、HashMap和ConcurrentHashMap的区别，HashMap的底层源码
先说HashMap与ConcurrentHashMap的区别：
第1点：HashMap不是线程安全的，ConcurrentHashMap是线程安全的
第2点：ConcurrentHashMap的执行效率比HashMap高，默认提高16倍
再说HashMap为什么不是线程安全的以及为什么ConcurrentHashMap是线程安全的。
因为HashMap的底层源码没有做任何线程安全的处理，自然就不是线程安全的了。ok，我们从底层的源码角度分析一下为什么？
说白了，HashMap底层就是一种散列结构，更通俗的说，就是数组+链表。为了添加一个元素到散列中，首先会使用hashCode函数（不过HashMap多hashCode函数做了特别处理）定位要插入的位置，再调用indexFor方法确定该元素要插入的桶的位置，之后呢？然后使用我们熟悉的equals方法比较，如果存在则覆盖，否则直接添加。

public V put(K key, V value) {
    if (table == EMPTY_TABLE) {
        inflateTable(threshold);
    }
    if (key == null)
        return putForNullKey(value);
    int hash = hash(key);
    int i = indexFor(hash, table.length);
    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
            V oldValue = e.value;
            e.value = value;
            e.recordAccess(this);
            return oldValue;
        }
    }
    modCount++;
    addEntry(hash, key, value, i);
    return null;
}
void addEntry(int hash, K key, V value, int bucketIndex) {
    if ((size >= threshold) && (null != table[bucketIndex])) {
        resize(2 * table.length);
        hash = (null != key) ? hash(key) : 0;
        bucketIndex = indexFor(hash, table.length);
    }
    createEntry(hash, key, value, bucketIndex);
}

在addEntry()方法中，首先会判断一下当前桶的大小，如果小于阈值*0.75的话就会扩大桶的容量，这样就可以使得添加的元素更加离散，从而降低哈希冲突的几率。

ConcurrentHashMap主要采用锁分段技术，对一个HashMap分成一个个Segment，对每个Segment执行加锁操作，以put为例，首先需要获取key的hashCode值，根据这个hashCode定位到具体的Segment，具体步骤是：将散列值右移 segmentShift 个位，并在高位填充 0 ，然后把得到的值与SegmentMask进行与操作获取Segment的下标，最后根据下标找到具体的Segment对象。在这个具体的Segment中执行put操作，首先会对要操作的Segment加锁，如果超过阈值就进行再哈希，把Segment中的table长度扩大一倍，否则把key的哈希值与table的长度-1进行“与”操作，得到该key在table中具体的索引值，根据这个索引值得到具体的桶，每一个桶存放的都是HashEntry对象，返回第一个HashEntry对象，执行循环，从该桶的位置开始遍历，如果存在则覆盖否则创建一个新的节点并放在链表的头部，添加完成后，count变量以及modCount的值增加1，最后释放锁。（注意：每个Segment的锁可以被同一个线程获取多次，但不可以被不同的线程获取）

12、TreeMap、HashMap、LinkedHashMap的区别
这个三个类都是Map接口的实现类，所以都具有Map接口的基本特性。
下面说说区别：
HashMap是最常用的类，允许使用nul作为key或者value，但是只能有一个null值的key，而可以对应多个null值的value，另外HashMap不支持线程的同步
LinkedHashMap会记录插入元素的顺序，使用迭代器遍历LinkedHashMap先输出的元素是先插入的元素，有HashMap的全部特性
TreeMap能够把保存的记录根据key排序，默认升序排序，TreeMap的key值以及value值不能为空

14、try catch finally，try里有return，finally还执行么？
如果try块中有return，由于finally值一定会执行的，所以在执行return之前会限制性finally块的代码，如果finally块没有return语句，则执行完finally块的代码后才只i系那个return语句。而如果finally块中也有return语句的话，则只会执行finally块的return，由于return之后整个方法结束，故不会执行try块的return语句。

先说Exception与Error包结构：
Exception包括Checked Exception与RuntimeException，RuntimeException常见的包括NullPointerException、ClassCastException、IndexOutOfBoundException等；Checked Exception包括IOException、SAXException、XMLParseException等。

16、Java面向对象的三个特征与含义
封装、继承与多态
封装：封装就是对外扩展开放，对内修改封闭，Java中的封装就是把数据以及对数据的操作放在一个个单元中（也就是类），对于其他使用该类的人来说，不需要关心其内部实现，只需要调用其方法就可以了
继承：就是在一个基础类的基础上创建新类的方法，创建的新类拥有基类的非private属性与方法，从而可以缩短开发周期
多态：分为设计时多态与运行时多态，设计时多态一般有重载，而对于覆盖或者继承的方法，Java运行时更具创建的实例决定调用哪个方法称为运行时多态。

17、Override和Overload的含义去区别
顾名思义，Override是重写的意思，重写发生在一个子类继承一个父类并重写父类的方法的时候，被重写的方法会覆盖父类的实现，当创建一个子类对象的时候，会调用被重写的方法，而不会调用其父类的方法。下面说Overload，就是重载的意思，重载是指具有相同方法名但具有不同参数类型的情况，但是返回值相同与否以及是否具有返回值不是判断两个函数是重载关系的根据。

18、Interface与abstract类的区别
第1点：一个类只能继承一个抽象类，但可以实现多个接口
第2点：抽象类可以有自己的数据成员，也可以有非抽象方法；接口可以有自己的数据成员，但必须是public static final的，且必须赋初值，接口中的方法都是public abstract的
第3点：实现一个抽象类或者接口必须实现其所有的抽象方法，即要么继续声明为abstract class要么实现其抽象方法
第4点：设计理念不同，抽象类是is-a的关系，而接口则是like-a的关系
第5点：抽象类可以没有抽象方法，但接口中必须是抽象方法

19、静态类与非静态类的区别
第1点：静态类不能被实例化，非静态类可以被实例化
第2点：静态类只包含静态属性、方法，非静态类既可以包含静态属性与方法又可以包含非静态属性与方法
第3点：静态类不能调用非静态的方法与属性，非静态类可以调用
第4点：静态方法只能被重载而不能被重写，因为静态类不需要创建实例对象

20、写出生产者消费者模式
问题背景：房子只能容纳100人 如果房子中没有人，是不会有人从门口出来的，如果房子中满了100人，外面的人需要等待屋子里的人走出门口才能进入

//问题背景
/*房子只能容纳100人 如果房子中没有人，是不会有人从门口出来的，如果房子中满了100人，外面的人需要等待屋子里的人走出门口才能进入*/
//进入的方法
public void enter() throws InterruptedException{
    //加锁
    lock.lock();
    try {
        while(persons == 100){
            full.await();
        }
        persons++;
        System.out.println("enter one,persons is " + persons);
        /*唤醒所有在等待的线程，不过仅仅是唤醒而已，还必须等当前线程释放锁后才能获得锁*/
        full.signalAll();
    } finally{
        lock.unlock();
    }
}

//出去的方法
public void exit() throws InterruptedException{
    //加锁
    lock.lock();
    try {
        while(persons == 0){
            empty.await();
        }
        persons--;
        System.out.println("exit one,persons is " + persons);
        //唤醒所有在等待的线程，不过仅仅是唤醒而已，还必须等当前线程释放锁后才能获得锁
        full.signalAll();
    } finally{
        lock.unlock();
    }
}

21、wait()和sleep()的区别
第1点：wait()方法是Object类的方法，sleep()是Thread类中的方法
第2点：wait()方法会释放锁资源，而sleep()方法不会释放锁资源。具体是这样的：调用wait方法的线程释放锁，其他线程可以通过竞争获得该锁，而该线程会进入等待池中，一般来说wait方法是没有时间限制的，因为这时资源已经被占用了，即使早一点结束等待状态也无济于事。只有其他线程执行了notify或者notifyAll方法后才被唤醒，从而进入就绪状态以竞争资源。sleep方法当等待的时间到了会自动唤醒
第3点：wait()、notify和notifyAll只能在同步方法或者同步块中使用，而sleep则没有这个限制

22、反射的作用与原理
反射是实现Java动态性的手段之一，反射机制允许我们在程序运行期间获取类的信息，只要给出类的名字就可以获得类的所有信息。
通过反射我们获取类的属性、方法、构造方法、静态方法、静态属性等信息，这些信息都存放在名为Class的对象中，每一个类都有一个Class对象，JVM在运行的时候也是从生成的.class文件中获取这个类的信息的，这里我们可以通过Class.forName()方法获取类的全部信息

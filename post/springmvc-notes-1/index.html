springmvc框架原理

如果你学过struts2的话，你会发现springmvc在很多处理流程上与struts2都相似，区别是springmvc基于方法开发，而struts2是基于类开发的（了解的小伙伴应该能马上想到ActionSupport类），下面来看看springmvc处理请求的具体过程：

  • Step 1：DispatcherServlet接收用户请求
  • Step 2：DispatcherServelt根据HandleMapping查找Handler(理解为Struts2中的处理器)
  • Step 3：HandlerMapping返回一条执行链（HandlerExecutionChain）
  • Step 4：DispatcherServelt让适配器执行Handler
  • Step 5：Handler把执行结果以ModelAndView的形式返回给适配器，DispatcherServlet接收ModelAndView对象
  • Step 6：DispatcherServelt把ModelAndView对象交给视图解析器解析ModelAndView中的View（顾名思义，ModelAndView包含Model对象和View对象，由于视图层只负责进行视图的呈现，故只需要其中的View对象即可）
  • Step 7：DispatcherServlet把视图解析器返回的View对象交给视图层进行视图渲染（这里的“渲染”可以理解为网页信息的还原）
  • Step 8：用户看到呈现结果

springmvc快速入门

环境准备

进行springmvc开发之前需要配置开发环境：

  • spring3.2全部的jar包
  • mysql 5.6
  • Myeclipse 2014
  • Jdk 1.7

开发配置

根据前面springmvc的框架，需要对项目进行一些配置：

1、前端控制器的配置（DispatcherServelt）
在web.xml中添加如下配置：

<!-- 前端控制器配置 -->
<servlet>
<servlet-name>springmvc</servlet-name>
<servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>

<!-- contextConfigLocation配置springmvc需要加载的组件 -->
<init-param>
<param-name>contextConfigLocation</param-name>
<param-value>classpath:springmvc.xml</param-value>
</init-param>
</servlet>

<servlet-mapping>
<servlet-name>springmvc</servlet-name>
<!-- 访问以.action结尾由DispathcerServlet文件 -->
<url-pattern>*.action</url-pattern>
</servlet-mapping>

2、视图解析器
在springmvc.xml中添加相关配置：

<!-- 视图解析器 -->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <!-- 配置前缀-->
    <property name="prefix" value="/WEB-INF/jsp/" />
    <!-- 配置后缀-->
    <property name="suffix" value=".jsp" />
</bean>

3、映射器配置

<bean class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping">
    <property name="mappings">
        <props>
            <prop key="/loginForm.action">loginController</prop>
        </props>
    </property>
</bean>

4、适配器配置

<!-- 非注解处理器适配器，所有的适配器都要 实现HandlerAdapter接口  执行handler-->
<bean class="org.springframework.web.servlet.mvc.SimpleControllerHandlerAdapter" />

5、处理器（Handler或者Controller）配置

<bean id="loginController" class="com.rhwayfun.ssm.controller.LoginController"></bean>

6、springmvc.xml的最终配置
springmvc.xml是springmvc的核心配置文件，与Struts2中的Struts.xml类似，根据上面的配置，我的springmvc.xml的最后是这样的（主要是多了配置文件的头部）：

<beans xmlns="http://www.springframework.org/schema/beans"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:mvc="http://www.springframework.org/schema/mvc"
xmlns:context="http://www.springframework.org/schema/context"
xmlns:aop="http://www.springframework.org/schema/aop" xmlns:tx="http://www.springframework.org/schema/tx"
xsi:schemaLocation="http://www.springframework.org/schema/beans 
    http://www.springframework.org/schema/beans/spring-beans-3.2.xsd 
    http://www.springframework.org/schema/mvc 
    http://www.springframework.org/schema/mvc/spring-mvc-3.2.xsd 
    http://www.springframework.org/schema/context 
    http://www.springframework.org/schema/context/spring-context-3.2.xsd 
    http://www.springframework.org/schema/aop 
    http://www.springframework.org/schema/aop/spring-aop-3.2.xsd 
    http://www.springframework.org/schema/tx 
    http://www.springframework.org/schema/tx/spring-tx-3.2.xsd ">

<!-- 配置处理器handler -->
<bean id="loginController" name="/loginAction.action" class="com.rhwayfun.ssm.controller.LoginController"></bean>

开发处理器（Handler）

业务需求：在界面显示演员列表

完成上面的工作后，就可以开发Handler了（controller），上面id为loginController的bean配置的就是处理器，class指定要处理请求为”/loginAction.action”的具体的类，所以我们需要开发LoginController类，处理该URL的请求：

public class LoginController implements Controller{

public ModelAndView handleRequest(HttpServletRequest request,
        HttpServletResponse response) throws Exception {
    List<Actor> actors = new ArrayList<Actor>();

    //人员列表
    Actor actor1 = new Actor();
    actor1.setUsername("李星云");
    actor1.setAge(23);
    actor1.setPassword("1234");

    Actor actor2 = new Actor();
    actor2.setUsername("姬如雪");
    actor2.setAge(22);
    actor2.setPassword("12334");

    actors.add(actor1);
    actors.add(actor2);

    //创建ModelAndView对象
    ModelAndView modelAndView = new ModelAndView();
    modelAndView.addObject("actors", actors);

    //设置视图
    //这里设置视图名应该为项目路径下的/WEB-INF/jsp/login/login.jsp，因为在上面的第2步中的视图解析器中配置了前缀和后缀，所以这里才能简写成现在的路径名
    modelAndView.setViewName("login/login");

    return modelAndView;
}

}

开发视图

视图层可以有多种技术实现，比如jsp、pdf、freemarker，这里使用jsp进行视图层的开发，根据业务逻辑，编写如下jsp页面：

<%@ page language="java" import="java.util.*" pageEncoding="utf-8"%>
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
<%@ taglib uri="http://java.sun.com/jsp/jstl/fmt"  prefix="fmt"%>
<%
String path = request.getContextPath();
String basePath = request.getScheme()+"://"+request.getServerName()+":"+request.getServerPort()+path+"/";
%>
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">
<html>
    <head>
        <base href="<%=basePath%>">
        <title>角色</title>
        <meta http-equiv="pragma" content="no-cache">
        <meta http-equiv="cache-control" content="no-cache">
        <meta http-equiv="expires" content="0">
        <meta http-equiv="keywords" content="keyword1,keyword2,keyword3">
        <meta http-equiv="description" content="This is my page">
    </head>
    <body>
<form action="${pageContext.request.contextPath }/login/loginForm.action" method="post">
角色列表：
<table width="100%" border=1>
<tr>
    <td>角色名称</td>
    <td>年龄</td>
    <td>密码</td>
</tr>
<c:forEach items="${actors }" var="actor">
<tr>
    <td>${actor.username }</td>
    <td>${actor.age }</td>
    <td>${actor.password}</td>
</tr>
</c:forEach>

</table>
</form>
    </body>
</html>

springmvc进阶

在上面的springmvc开发配置中，使用的是非注解的方式，所以整个配置文件看起来比较臃肿，下面使用springmvc的基于注解的开发方式：

配置基于注解的映射器

<!--注解处理器映射器 -->
<bean class="org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerMapping"/>

配置基于注解的适配器

<!--注解处理器适配器 -->
<bean class="org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerAdapter"/>

开发Handler

@Controller
public class LoginController2{

//@RequestMapping实现url与loginForm2方法进行映射，一个方法对应url
@RequestMapping("/loginForm2")
public ModelAndView handleRequest(HttpServletRequest request,
        HttpServletResponse response) throws Exception {
    List<Actor> actors = new ArrayList<Actor>();

    //人员列表
    Actor actor1 = new Actor();
    actor1.setUsername("李星云");
    actor1.setAge(23);
    actor1.setPassword("1234");

    Actor actor2 = new Actor();
    actor2.setUsername("姬如雪");
    actor2.setAge(22);
    actor2.setPassword("12334");

    actors.add(actor1);
    actors.add(actor2);

    //创建ModelAndView对象
    ModelAndView modelAndView = new ModelAndView();
    modelAndView.addObject("actors", actors);

    //设置视图
    modelAndView.setViewName("login/login");

    return modelAndView;
}

}

加载Handler

由于使用的是基于注解的开发，需要让spring容器识别注解的Handler对象，因此，需要在springmvc.xml文件中添加如下的配置：

<!-- 使用组件扫描的方式配置handler -->
<context:component-scan base-package="com.rhwayfun.ssm.controller"></context:component-scan>
